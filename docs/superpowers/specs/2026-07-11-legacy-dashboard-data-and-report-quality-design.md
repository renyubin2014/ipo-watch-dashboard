# 旧版 IPO 看板数据与报告质量改造设计

## 目标

在不改变旧版首页布局的前提下，统一当前 15 家公司的发布数据，修复测试与 Python 环境，并提升“快速看懂”和“完整调研”的分析质量、可追溯性与表达一致性。

后续只通过 GitHub Pages 更新。Sites 暂停使用。本次不修改项目中的周调度任务，也不负责企业池的周期更新。

## 基准与范围

- 当前发布集合以旧版 GitHub Pages 已发布的 15 家公司为基准。
- 长鑫科技保留在发布集合中。
- 当前本地数据额外出现的天海汽车电子、南网数字、洛阳盛龙矿业、双欣环保不纳入本次发布集合。
- 周调度、官方状态扫描、公司进入或退出企业池的逻辑不在本次范围。
- 旧版根首页、电脑版看板和手机版看板的页面结构与视觉布局不改。
- 不恢复或继续开发已删除的 V2 看板。

## 单一数据源

规范化后的 `dashboard/data.js` 是当前发布集合和公司展示字段的唯一发布数据源。

`dashboard/reports.js` 只负责为同一批公司提供报告路径与报告元信息，不得形成独立公司池。电脑版、手机版和 GitHub Pages 发布包都直接消费同一份 `dashboard/data.js` 与 `dashboard/reports.js`。

发布质量门必须验证：

1. `data.js` 恰好包含当前基准的 15 家公司。
2. `reports.js` 的公司集合与 `data.js` 完全相同，不多也不少。
3. 每家公司同时存在“快速看懂”和“完整调研”报告。
4. 两类报告路径均可解析到实际文件。
5. 发布包中的数据、映射和源目录一致。

任何一项失败都阻止构建和发布，不能通过页面层补丁掩盖分叉。

## Python 可复现环境与测试

项目明确锁定 Python 版本和直接依赖版本，至少包含报告解析所需的 `pypdf`。开发者按项目说明创建隔离环境后，应能用一条固定命令运行全部测试。

修复现有测试暴露的问题：

- 18 家数据与 15 家报告映射不一致。
- 嘉立创板块字段未规范为“深交所主板”。
- 系统 Python 缺少 `pypdf` 导致测试模块无法导入。

新增测试覆盖单一数据源、报告映射完整性、发布包一致性以及报告来源质量门。测试不依赖周调度任务执行。

## 完整调研来源列表质量门

PDF、状态、单位、页码和报告映射的质量信息只体现在“完整调研”的来源列表及其构建校验中，不加入首页和“快速看懂”。

完整调研的来源列表中，每条用于关键数据的来源至少包含：

- 来源级别与文件名称。
- 官方链接或本地 PDF 映射。
- 文件真实性检查结果，确认正文为有效 PDF，而不是 HTML、错误页或空文件。
- 公司状态及状态核验日期。
- 对应数据的期间和单位。
- PDF 页码或明确章节。
- 该来源用于报告中的章节或指标。
- 镜像、媒体转述、研究口径或待复核限制。

关键财务数据缺少单位、期间或页码时，报告生成失败；确实无法取得的字段必须明确标记“待复核”，且不得包装成官方确定事实。

## 快速看懂

保留长鑫科技概览页的整体章节、组件和表达节奏。其他公司按行业实际情况学习其结构，不机械复制半导体内容。

在现有正文开头增加“先说结论”模块，放在首屏信息之后、其他分析章节之前。模块重点回答：

- 最大优势：1—3 项，每项用通俗的一句话讲清优势是什么、为什么成立、最关键证据是什么。
- 最大风险：1—3 项，每项用通俗的一句话讲清风险如何发生、影响什么、首先观察什么信号。

优势和风险必须针对具体公司，不允许只写“行业前景广阔”“竞争激烈”等模板话。概览页所用事实必须能在对应完整调研的来源列表中追溯。

## 完整调研新增分析模块

完整调研沿用长鑫科技详细版的章节层级、粘性目录、事实标签、风险矩阵和来源列表表达，并加入三类通俗分析：

### 利润机制

解释公司怎样赚钱，以及收入如何转化为利润。至少覆盖价格、销量或产能、成本、毛利率、费用、现金流中的关键杠杆，并指出哪个变量最容易让利润发生非线性变化。

### 非共识判断

给出 1—3 个可能不同于常见市场叙事、但有证据链支持的判断。每个判断必须分开写明事实依据、推理过程和仍然缺失的证据，不能把主观判断写成事实。

### 反证信号

每个核心判断都应给出可观察的反证条件，说明出现什么公告、财务指标、客户变化、价格变化或项目进度时，需要下调或推翻原判断。

## 发布与验收

实施完成后只更新原 GitHub Pages 仓库，不使用 Sites。

验收标准：

1. 旧版根首页、电脑版和手机版布局文件没有非必要结构变化。
2. 当前发布集合稳定为 15 家，数据与报告映射集合完全一致。
3. 15 家都能打开“快速看懂”和“完整调研”。
4. 全部自动化测试在锁定的 Python 环境中通过。
5. 完整调研来源列表通过 PDF、状态、单位、页码和报告映射质量门。
6. 15 家快速看懂都包含具体的最大优势与最大风险总结。
7. 15 家完整调研都包含利润机制、非共识判断和反证信号。
8. 桌面端与 390px 手机版无横向溢出、文字压叠和交互遮挡。
9. GitHub Pages 发布包构建成功，线上旧版入口和报告链接可访问。
