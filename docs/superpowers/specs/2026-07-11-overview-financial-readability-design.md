# 快速看懂财务可读性优化设计

## 范围

优化当前15家公司“快速看懂”页面，不修改首页布局、完整调研结构、企业池和周调度任务。

## 问题根因

1. 关键事实卡直接复用数据层长句，导致多个年度数字挤在一张卡内，分隔不清甚至发生数字粘连。
2. 标准化脚本整体复制原章节，使行业位置图和图下明细表重复表达同一信息。
3. 财务读数没有独立的数据合同，单位、精度、期间及实际值和预计值的层级不统一。
4. 快速看懂没有优先使用完整调研中已经披露且可追溯的2026年最新期间数据。

## 页面合同

### IPO与关键事实

- 每张卡只表达一个主事实，不再放三年数字长串。
- 主值格式为“期间｜数值”，脚注补充同比、口径或判断。
- 最新收入和最新利润优先级：2026年实际数据 > 2026年公司预计 > 2025年度数据。
- 公司预计必须明确标注“预计”，不得与实际数据混用。
- 金额统一为万元、整数和千分位；百分比及倍数保留两位小数。

### 公司与行业位置

- 保留最能解释公司位置的一张核心图。
- 删除与图重复的明细表。
- 保留图表标题、读图结论和必要口径说明。
- 不删除图中不可替代的对比数据。

### 财务读数

- 标题区明确标注“单位：万元；百分比除外”。
- 2023—2025年度数据作为历史趋势，金额取整。
- 有可靠披露时增加2026Q1实际数据、2026H1预计数据等最新期间。
- 实际数据与预计数据使用不同标签和视觉样式。
- 每个指标按“指标名、期间、主值、同比或解释”分层展示，避免长句。
- 若没有2026年数据，明确写“最新披露仍为2025年度”，不推算、不编造。

## 数据与追溯

- 概览只能使用当前完整调研中已经存在且能在来源列表追溯的数据。
- 2026年数据逐家公司核对最新招股书、审计截止日后事项和盈利预测章节。
- 对预计区间保留上下限及“公司预计”标签。
- 不确定或无法稳定提取的数据继续标为待补充，不降级使用媒体转述。

## 15家公司检查

逐家检查：

1. 关键事实卡是否存在数字粘连或年度长串。
2. 行业位置是否存在图和表重复。
3. 财务读数是否明确单位并使用整数万元。
4. 是否遗漏完整调研已有的2026年实际或预计数据。
5. 移动端是否横向溢出、遮挡或压叠。

## 质量门

- 自动化测试锁定关键事实的短格式、金额单位和精度。
- 自动化测试禁止行业位置图后出现重复数据表。
- 自动化测试检查2026年数据存在时必须进入快速看懂。
- 15家公司分别在390、768、1280像素宽度完成浏览器检查。
- 完整测试和报告质量门通过后才能更新GitHub。
