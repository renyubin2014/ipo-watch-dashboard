# 报告结论样式与响应式修复设计

## 目标

在不改变旧版看板整体布局的前提下，完成两项改造：

1. 将“最大优势 / 最大风险”改为用户选定的 B 方案——左侧色条结论。
2. 修复电脑版、手机版看板及30份公司报告在不同屏幕宽度下的内容溢出和异常排版。

后续仍只发布到 GitHub Pages，不使用 Sites，不修改周调度任务。

## 优势与风险样式

“快速看懂”的“先说结论”模块采用纵向结论条：

- 最大优势使用低饱和绿色细条。
- 最大风险使用低饱和红色细条。
- 每条包含标题、通俗结论、成立原因或风险机制、观察信号。
- 多项优势或风险按重要性纵向排列，不在窄屏强制并排。
- 电脑端保持研究报告感，不做营销式大色块。
- 手机端保持单列，正文宽度随容器收缩。

完整调研中的风险矩阵不改成同一组件，避免全页视觉重复；但核心结论与反证信号继续保持对应关系。

## 已确认的溢出根因

- 完整调研的 `.layout` 在手机端虽然切换成单列，但内部表格的最小内容宽度约为931px，反向撑宽整个网格列。
- 网格和弹性布局的直接子项缺少 `min-width: 0`，无法小于内容固有宽度。
- 官方链接、PDF文件名、英文接口名和连续字符缺少安全断行规则。
- 手机版板块导航内容宽度约428px，390px屏幕下最后一个导航项被截断，且可滑动提示不清楚。
- 长鑫概览中的滑块行存在约2px的内部越界。

## 响应式修复

### 看板

- 旧版电脑首页结构与视觉布局不变。
- 电脑版在768px和1280px下保持现有卡片组织方式。
- 手机版板块导航保留横向滑动，但增加右侧渐隐提示和足够的末端内边距，最后一项可完整滚动到可视区域。
- 公司名、状态、按钮和说明文字允许安全换行，不覆盖相邻元素。
- 页面整体禁止横向滚动。

### 快速看懂

- 所有网格直接子项设置可收缩边界。
- 长链接、路径、英文缩写和连续文本使用安全断行。
- 图表、滑块、标签和卡片宽度不超过父容器。
- 选定的 B 样式在桌面端和手机端都使用纵向排列。

### 完整调研

- `.layout`、内容列、卡片和章节设置 `min-width: 0`。
- 手机端目录改成横向滚动的标签栏，避免占据过长首屏。
- 表格在手机端进入独立横向滚动容器或等价的局部滚动模式；只有表格内部允许横向滑动，不能撑宽页面。
- 来源路径、PDF文件名、官方链接和接口名称使用 `overflow-wrap: anywhere`。
- 普通正文、标题、标签、图表与交互组件必须保持在视口内。

## 实现边界

- 优先修改报告生成模板和升级脚本，避免只给当前HTML打一次性补丁。
- 同步更新当前15家公司的30份报告。
- 首页布局文件仅做响应式缺陷修复，不重排桌面结构。
- 不修改企业池、数据内容、周调度、Sites或V2。

## 测试与验收

自动化测试必须覆盖：

- B 样式的优势和风险模块存在且顺序正确。
- 响应式基础规则包含网格收缩、长文本断行、表格局部滚动和移动目录。
- `dashboard/data.js` 与 `dashboard/reports.js` 仍为同一15家公司。
- 30份报告链接全部存在。

浏览器验收宽度：

- 390px：主流手机窄屏。
- 768px：平板或窄窗口。
- 1280px：电脑端。

逐页验收范围：

- 电脑版看板。
- 手机版看板。
- 15份“快速看懂”。
- 15份“完整调研”。

通过标准：

- `document.documentElement.scrollWidth <= clientWidth`。
- 普通正文、标题、卡片、导航和交互组件不越界。
- 表格若超宽，只能在表格自身区域滑动。
- 390px下无文字压叠、按钮遮挡或最后导航项不可达。
- 1280px下旧版看板结构与视觉节奏不发生非必要变化。
