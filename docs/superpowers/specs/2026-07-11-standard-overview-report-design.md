# 快速看懂标准化设计

## 目标

统一当前15家公司的“快速看懂”结构、数据格式和页面节奏，消除文字被卡片遮挡、首屏文字过量、章节顺序漂移、单位不统一以及概览页重复展示来源列表的问题。

本次只修改“快速看懂”及其生成模板；完整调研保留现有来源质量门。企业池、周调度、Sites和V2不在范围内。

## 已确认根因

- 新增的“先说结论”被插入原指标卡之前，但原指标卡保留负上边距，导致卡片向上覆盖结论文字。
- 不同报告来自不同模板，Hero、指标卡、结论和章节的先后顺序不一致。
- 部分Hero副标题同时承载公司介绍、数据摘要、来源限制和研究结论，形成大段文字。
- 当前15份概览均保留来源列表，与“快速看懂”的阅读目标重复。
- 金额同时存在亿元、亿和万元等单位，小数位也不统一。

## 统一页面结构

15份快速看懂必须按以下顺序输出，不能因公司或原模板不同而改变：

1. **Hero背景图首屏**
   - 公司名称。
   - 上市板块与IPO状态标签。
   - 一句话主营业务。
   - 一句话核心矛盾或最重要验证点。
   - Hero正文最多两句话，不放来源说明、数据限制、长财务摘要或分析过程。

2. **先说结论**
   - 紧接Hero，不允许其他指标卡或长段落插入其前。
   - 使用已选定的B样式：纵向结论条、左侧低饱和色条。
   - 最大优势在前，最大风险在后。
   - 每条包含明确结论和通俗解释，不写模板化空话。

3. **IPO与关键事实**
   - IPO状态。
   - 拟募资金额。
   - 最新一期营业收入。
   - 最新一期利润指标，必须明确是净利润、归母净利润或扣非归母净利润。
   - 最重要待确认事项。

4. **公司与行业位置**

5. **产品或业务阶梯**

6. **上下游关系**

7. **财务读数**

8. **后续重点跟踪**

概览页末尾不显示来源列表。所有概览数据必须仍能在对应完整调研的来源列表中追溯。

## 数据格式

快速看懂中的金额统一遵循：

- 单位固定为“万元”，不转换为亿元。
- 四舍五入到整数。
- 使用千位分隔符。
- 示例：`2,950,000万元`、`99,016万元`、`-116,400万元`。
- 同一金额在Hero、事实卡、图表和正文中重复出现时，格式必须完全一致。

其他数据格式：

- 百分比统一保留两位，例如 `42.89%`。
- 倍数统一保留两位，例如 `0.79倍`。
- 排名、状态、日期、页码和数量保持自然整数表达，不补无意义小数。
- 待补充或待复核字段不转换为数字。

## 排版与遮挡修复

- “先说结论”必须占据正常文档流，后续指标卡不得用负边距覆盖它。
- 负边距视觉效果仅允许用于Hero与其直接相邻的事实卡；采用标准结构后，结论位于Hero后，事实卡不得再向上覆盖结论。
- 所有结论条、事实卡、图表和章节必须设置可收缩边界。
- 长公司名、文件名和连续字符必须安全换行。
- 390px、768px和1280px下不得出现文字裁切、卡片重叠或页面级横向溢出。

## 模板与升级策略

- 报告生成模板直接输出统一章节顺序，不能仅依赖生成后的临时补丁。
- 当前15份报告通过升级脚本迁移到同一结构，并生成新版本文件，不覆盖旧报告。
- 升级脚本必须能识别长鑫、燧原、宇树和通用模板的不同旧结构，抽取现有模块后按统一顺序重组。
- 现有公司专属图表和分析内容可以保留，但必须放入对应的标准章节槽位。
- 无法归类的旧模块不能堆在Hero与“先说结论”之间；应放入最接近的标准章节或从概览移除。

## 测试与验收

自动化合同：

- 15份快速看懂均存在且仅存在一个“先说结论”。
- Hero到“先说结论”之间没有事实卡、表格或长分析段落。
- 8个标准章节按规定顺序出现。
- 快速看懂不包含“来源列表”章节。
- 金额不出现“亿元”或裸“亿”单位。
- 金额使用万元整数格式，不保留金额小数。
- 百分比和倍数保留两位。

浏览器验收：

- 检查15份快速看懂的390px、768px和1280px，共45次页面审计。
- 检查文本元素与卡片的几何重叠。
- 检查页面 `scrollWidth <= clientWidth`。
- 检查Hero正文不超过两段，且“先说结论”紧接Hero。
- 检查概览页所有表格和图表在窄屏内可读。
